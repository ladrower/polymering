OpenGL workbook | code source

Obiekt programu zawiera wszystkie informacje niezbedne do umieszenia zbioru jednego lub wiekszej liczby obiektow shaderow w programowalnej czesci potoku renderingu OpenGL

Obiekt programu tworzy wywolanie funkcji

GLuint glCreateProgram( void );

Obiekt programu moze byc usunienty przy uzyciu funkcji

void glDeleteProgram( GLuint program );

Dolaczenie obiektu shadera do obiektu programu wymaga uzycia funkcji

void glAttachShader( GLuint program, GLuint shader );

Mozliwe jest dolaczenie wielu obiektow shaders obslugujachych programy cieniowania tego samego typu (np. shaderow wierzcholkow) do jednego obiektu programu,
a ponadto jeden obiekt shadera moze byc dolaczony do wiecej niz jednego obiektu programu.

Aby odlaczyc obiekt shaders z oobiektu programu, nalezy uzyc funkcji

void glDetachShader( GLuint program, GLuint shader );

Po dolaczeniu wybranych obiektow shaders nalezy dokonac konsolidacji obiektu programu, ktora realizuje funkcja

void glLinkProgram( GLuint program );

Wynik procesu konsolidacji przechowywany jest w zmiennej stanu obiektu programu GL_LINK_STATUS
Wartosc tej zmiennej mozna odczytac, korzystajac z opisanej dalej funkcji glGetProgramiv

Opcjonalnym etapem przetwarzania obiektu programu jest sprawdzenie jego poprawnosci przy uzyciu funkcji

void glValidateProgram( GLuint program );

ktorej parametr program zawiera identyfikator weryfikowanego obiektu programu.
Wynik sprawdzania zawiera zmienna stanu obiektu programu GL_VALIDATE_STATUS, ktora moze byc pobrana przy uzyciu funkcji glGetProgramiv

Gdy obiekt programu zostal pomyslnie skonsolidowany i opcjonalne zweryfikowany, moze sie on stac czescia biezacego potoku renderingu poprzez wywolanie funkcji

void glUseProgram( GLuint program );

ktorej parametr program jest identyfikatorem obiektu programu.
Podanie wartosci 0 spowoduje wylaczenie biezacego obiektu programu z potoku renderingu.
